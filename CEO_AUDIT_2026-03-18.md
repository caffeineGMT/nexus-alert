## 🔴 P0 - CRITICAL BUGS (Fix Within 24 Hours)

### 1. Broken Chrome Web Store Links
**Issue:** All CTA buttons link to `EXTENSION_ID` placeholder instead of actual Chrome Web Store ID
**Impact:** 100% of users clicking "Install" buttons get 404 errors - **ZERO conversions possible**
**Files Affected:** 67+ instances across `web/src/app/page.tsx`, `global-entry/page.tsx`, `sentri/page.tsx`, all blog posts
**Fix Required:**
```bash
# Find actual Chrome Web Store ID and replace all instances
find web/src/app -type f -name "*.tsx" -exec sed -i '' 's|EXTENSION_ID|ACTUAL_CHROME_STORE_ID|g' {} +
```

### 2. Error Tracking Completely Disabled
**Issue:** Sentry DSN is hardcoded placeholder `REPLACE_WITH_YOUR_SENTRY_DSN`
**Impact:** Zero production error visibility - bugs go undetected until users report them
**File:** `background.js:9`
**Fix Required:**
- Sign up for Sentry (free tier)
- Add actual DSN to background.js
- Add DSN to backend worker.js
- Verify error capture in Sentry dashboard

### 3. Missing Social Media OG Image
**Issue:** `web/src/app/layout.tsx:32` has TODO for OG image - using placeholder
**Impact:** Poor social media engagement - no visual preview when shared on Facebook/Twitter/LinkedIn
**Fix Required:**
- Design 1200x630 OG image with branding
- Replace `/og-image.png` in web/public/
- Test with Facebook Debugger and Twitter Card Validator

## 🟠 P1 - HIGH PRIORITY (Fix Within 1 Week)

### 4. Catastrophic Accessibility Failures
**Issue:** Only 3 ARIA labels in entire codebase - fails WCAG 2.1 AA standards
**Impact:** Screen reader users cannot use the product - ADA lawsuit risk
**Fix Required:**
- Add `aria-label` to all interactive elements
- Add `role` attributes to semantic sections
- Add `alt` text to ALL images (currently 0)
- Test with VoiceOver (macOS) and NVDA (Windows)
- Target: 100+ accessibility attributes minimum

### 5. Production Debug Code
**Issue:** 6 `console.log()` statements in production code
**Impact:** Performance degradation, exposes internal logic to competitors
**Files:** `background.js`, `popup.js`
**Fix Required:**
```bash
# Remove all console.log statements
grep -rl "console\.log" background.js popup.js | xargs sed -i '' '/console\.log/d'
```

### 6. Minimal Error Handling
**Issue:** Only 4 try/catch blocks across critical extension code
**Impact:** Unhandled exceptions crash the extension, user sees blank popup
**Fix Required:**
- Wrap all async operations in try/catch
- Add error boundaries in popup.js
- Display user-friendly error messages instead of crashes
- Target: 20+ try/catch blocks minimum

### 7. No Image Optimization
**Issue:** Zero WebP/AVIF images, zero lazy loading attributes
**Impact:** Slow page load (LCP >3s on 3G), poor SEO ranking
**Fix Required:**
- Convert all PNG/JPG to WebP with fallbacks
- Add `loading="lazy"` to all below-fold images
- Add `fetchPriority="high"` to hero image
- Use Next.js `<Image>` component for automatic optimization

### 8. Large Bundle Sizes
**Issue:**
- Backend worker.js: 119KB (Cloudflare limit is 1MB, but best practice is <100KB)
- CSS bundle: 65KB (should be <20KB for critical path)
**Impact:** Slow cold starts on Cloudflare Workers, slow page paint
**Fix Required:**
- Code split worker.js into modules
- Remove unused CSS with PurgeCSS
- Enable Brotli compression
- Lazy load non-critical components

### 9. Mobile Onboarding Breaks
**Issue:** `onboarding.html:24` has `min-width: 480px` - breaks on iPhone SE (375px)
**Impact:** 15% of users on small phones see horizontal scroll and broken layout
**Fix Required:**
```css
/* Change to responsive */
body {
  min-width: 320px; /* iPhone SE support */
  max-width: 640px;
  padding: 20px 16px;
}
@media (max-width: 480px) {
  .container { padding: 24px 16px; }
}
```

### 10. Zero UI Test Coverage
**Issue:** No `data-testid` attributes anywhere - impossible to write E2E tests
**Impact:** No automated QA - regressions slip into production
**Fix Required:**
- Add `data-testid` to all buttons, forms, critical UI elements
- Set up Playwright E2E tests
- Target: 80% coverage of critical user flows

## 🟡 P2 - MEDIUM PRIORITY (Fix Within 2 Weeks)

### 11. No PWA Support
**Issue:** Missing `manifest.json` for Progressive Web App
**Impact:** Mobile users can't install as standalone app - lower retention
**Fix Required:**
- Create `/web/public/manifest.json`
- Add theme colors, icons, display mode
- Add service worker for offline support
- Enable "Add to Home Screen" on mobile

### 12. Limited Code Splitting
**Issue:** Only 15 components use React.lazy/Suspense
**Impact:** Large initial bundle size (>500KB) - slow Time to Interactive
**Fix Required:**
- Split all below-fold components
- Lazy load pricing section, testimonials, blog posts
- Use Next.js dynamic imports
- Target: <200KB initial bundle

### 13. No Keyboard Navigation
**Issue:** Extension popup has no keyboard shortcuts
**Impact:** Power users can't navigate efficiently - poor UX
**Fix Required:**
- Add tab index to all interactive elements
- Implement keyboard shortcuts:
  - `Space` = Toggle monitoring on/off
  - `Arrow keys` = Navigate locations
  - `Enter` = Select location
  - `Esc` = Close popup
- Add keyboard shortcut help modal (`?` key)

### 14. Missing Dark/Light Mode Toggle
**Issue:** Hardcoded dark theme - no user preference
**Impact:** Users in bright environments struggle to read (accessibility issue)
**Fix Required:**
- Add theme toggle button in settings
- Store preference in chrome.storage
- Support system preference `prefers-color-scheme`
- Update all color variables for light theme

### 15. No Timezone Clarity
**Issue:** All times shown in user's local timezone without indication
**Impact:** Confusion when sharing screenshots, support tickets harder to debug
**Fix Required:**
- Display timezone abbreviation next to times (e.g., "2:30 PM PST")
- Add tooltip showing UTC time on hover
- Store all times as UTC internally

## 🟢 P3 - NICE TO HAVE (Fix Within 1 Month)

### 16. No Slot History Export
**Issue:** Users can't export their slot history as CSV/JSON
**Impact:** Users lose data when reinstalling extension - poor retention
**Fix Required:**
- Add "Export History" button in Settings tab
- Generate CSV with columns: Date, Location, Program, Timestamp
- Auto-download file
- Bonus: Add "Import History" for migration

### 17. No Custom Notification Sounds
**Issue:** Users stuck with default beep sound
**Impact:** Users in shared spaces can't distinguish notifications
**Fix Required:**
- Add sound picker in settings
- Include 5 preset sounds (chime, bell, ping, alert, silent)
- Allow custom sound upload (MP3/WAV, max 100KB)
- Preview sound before saving

### 18. No Offline Support
**Issue:** Extension requires internet connection to show UI
**Impact:** Users on planes/trains can't review past slot history
**Fix Required:**
- Cache last known state in chrome.storage.local
- Show cached data when offline
- Display "Offline Mode" banner
- Sync when connection restored

### 19. Missing A/B Test Dashboard
**Issue:** Price variant tracking exists but no admin dashboard
**Impact:** Can't analyze conversion rates by price point - flying blind on pricing optimization
**Fix Required:**
- Create `/web/src/app/admin` route
- Show conversion rates by variant (control vs test)
- Calculate statistical significance
- Add charts with Chart.js or Recharts
- Protect with basic auth

### 20. No Churn Prediction
**Issue:** No proactive retention for at-risk users
**Impact:** Users churn silently - no chance to win them back
**Fix Required:**
- Track engagement signals:
  - Last active timestamp
  - Monitoring toggle frequency
  - Settings changes
- Identify at-risk users (no activity >7 days)
- Send re-engagement email with discount code
- Show exit survey on uninstall (Chrome Web Store API)
