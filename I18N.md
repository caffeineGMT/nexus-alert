# Internationalization (i18n) for NEXUS Alert

This document explains the internationalization system for both the Chrome extension and web application.

## Supported Languages

- **English (en)** - Default language
- **Spanish (es)** - For Spanish-speaking Canadian users
- **French (fr)** - For French-speaking Canadian users

---

## Chrome Extension i18n

### Structure

The Chrome extension uses the native `chrome.i18n` API for internationalization.

```
_locales/
├── en/
│   └── messages.json    # English translations
├── es/
│   └── messages.json    # Spanish translations
└── fr/
    └── messages.json    # French translations
```

### How It Works

1. **Manifest Integration**: The `manifest.json` uses `__MSG_extName__` and `__MSG_extDescription__` for localized extension metadata.

2. **Automatic Detection**: Chrome automatically detects the user's browser language and loads the appropriate locale.

3. **i18n Helper**: The `popup.js` includes an i18n helper that:
   - Translates elements with `data-i18n` attributes on page load
   - Provides `i18n.getMessage(key)` function for dynamic translations

### Adding Translations to Extension

1. **For HTML elements**: Add a `data-i18n` attribute with the message key:
   ```html
   <button data-i18n="addLocation">+ Add Location</button>
   ```

2. **For JavaScript strings**: Use the `i18n.getMessage()` function:
   ```javascript
   const message = i18n.getMessage('errorBannerCBP');
   showToast(message, 'error');
   ```

3. **Add new translation keys**: Edit all three files in `_locales/*/messages.json`:
   ```json
   {
     "keyName": {
       "message": "Translated text here"
     }
   }
   ```

## Web Application i18n

### Structure

The web application uses a client-side i18n system compatible with static export:

```
web/src/
├── messages/
│   ├── en.json          # English translations
│   ├── es.json          # Spanish translations
│   └── fr.json          # French translations
├── hooks/
│   └── useTranslations.ts   # Translation hook
└── components/
    └── LanguageSwitcher.tsx  # Language selector
```

### How It Works

1. **Client-Side Translation**: Uses localStorage to persist language preference
2. **Automatic Detection**: Detects browser language on first visit
3. **No Page Reload**: Language changes reload the page to apply new translations

### Using Translations in Components

1. **Import the hook**:
   ```typescript
   import { useTranslations } from '@/hooks/useTranslations';
   ```

2. **Use in your component**:
   ```typescript
   export default function MyComponent() {
     const { t } = useTranslations();

     return (
       <div>
         <h1>{t('hero.title')}</h1>
         <p>{t('hero.description')}</p>
       </div>
     );
   }
   ```

3. **Access nested translations**:
   ```typescript
   t('nav.brand')              // "NEXUS Alert"
   t('hero.titleHighlight')    // "NEXUS Appointment Tracker"
   t('pricing.free.name')      // "Free"
   ```

### Language Switcher

The `<LanguageSwitcher />` component is already included in the navigation bar. It:
- Shows current language with flag emoji
- Allows users to switch between English, Spanish, and French
- Persists preference in localStorage
- Reloads page to apply new language

### Adding New Translations

1. **Edit translation files**: Add your key to all three files:

   **en.json**:
   ```json
   {
     "mySection": {
       "title": "My Title",
       "description": "My description"
     }
   }
   ```

   **es.json**:
   ```json
   {
     "mySection": {
       "title": "Mi Título",
       "description": "Mi descripción"
     }
   }
   ```

   **fr.json**:
   ```json
   {
     "mySection": {
       "title": "Mon Titre",
       "description": "Ma description"
     }
   }
   ```

2. **Use in components**:
   ```typescript
   const { t } = useTranslations();
   <h1>{t('mySection.title')}</h1>
   ```

## Translation Guidelines

### For Canadian Market

1. **Spanish (es)**:
   - Use neutral Latin American Spanish
   - Avoid regional slang or colloquialisms
   - Use "tú" form (informal but respectful)

2. **French (fr)**:
   - Use Canadian French conventions
   - Follow Quebec French spelling when different from European French
   - Maintain formal tone for official content

### Best Practices

1. **Keep keys semantic**: Use descriptive key names like `hero.title` instead of `text1`
2. **Maintain consistency**: Use the same terminology across all translations
3. **Test all languages**: Switch languages to verify translations display correctly
4. **Avoid hardcoded text**: Always use translation keys, never hardcode strings
5. **Handle plurals carefully**: Consider using different keys for singular/plural forms
